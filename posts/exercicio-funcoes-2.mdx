## Introdução

Como vimos, funções são estruturas das linguagens de programação que permitem o **encapusalamento** de **abstrações** e **conceitos**. Ao criar uma função, permitimos que aquele trecho de código seja reaproveitado em outras partes do programa ou até **em outros programas** caso esta seja exportada na forma de uma **biblioteca**.

Assim, temos que **biblioteca** é uma coleção de implementações de abstrações com interface bem definida, em uma dada linguagem. Ou de maneira mais simplória, você pode pensar que uma biblioteca é um conjunto de arquivos de código que, após a instalação, estão disponíveis para serem utilizados em seu programa. Bastando importar a biblioteca no corpo do programa através do comando *import*. Ex:

```python
import random
import time
import datetime
```

Existem bibliotecas que fazem parte do núcleo da linguagem. Elas são instaladas juntamente com o interpretador da linguagem por proverem funcionalidades muito básicas e/ou por serem utilizadas em grande parte dos programas. Por exemplo, a função *print* que imprime um texto na tela.

Dado que estas bibliotecas farão parte de todos os programas, elas são cuidadosamente definidas e implementadas pelo comitê que gerencia a linguagem afim de evitar que o núcleo da linguagem cresça demais em tamanho. Assim, é dado um ferramental básico ao programador, que pode instalar outras bibliotecas que o ajude a resolver o problema em questão.

Cada linguagem possui um ou mais repositórios através dos quais é possível compartilhar bibliotecas. O repositório oficial de Python é o [PyPI](https://pypi.org/). Dê uma olhada nele se quiser, mas isso é papo para outra aula.

Por hora, vamos focar no uso de bibliotecas presentes no núcleo da linguagem.


## Exercício 1: O jogo de adivinhar

Implemente um jogo no qual o usuário tenta, interativamente, adivinhar um número escolhido aleatoriamente pelo computador. O jogo deve pedir um novo *chute* até que o usuário acerte o número escolhido.

Afim de facilitar o acerto, o número escolhido pelo computador deve ser um inteiro entre 1 e 99.

Para gerar o número aleatório, utilize o fragmento de código abaixo que gera um número inteiro entre 1 e 99 através função *randint* da biblioteca *random*.

```python
import random

def generateRandomNumber():
    return random.randint(1, 99)
```

Exemplo de execução do programa:

```
% python guessing_game.py

###### The Guessing Game ######

Enter an integer from 1 to 99: 50
guess is low
Enter an integer from 1 to 99: 75
guess is high
Enter an integer from 1 to 99: 60
guess is high
Enter an integer from 1 to 99: 55
guess is low
Enter an integer from 1 to 99: 57
Congratulations! You guessed it!
```

## Exercício 2: Chute estratégico

Pro jogo ficar mais divertido, vamos colocar um limite de *chutes*. Assim, o usuário tem que ser um pouco mais estratégico na hora de *chutar* (ou contar com grande sorte).

Implemente um limite de 5 chutes por partida.

Consegue adivinhar o número agora?

## Exercícios extras

Quem é o *adivinhão* da sua turma? Vamos deixar o jogo mais competitivo adicionando um placar com o número de *chutes* por partida. Para isso:
- adicione um menu inicial em que o usuário possa escolher entre jogar mais uma partida, mostrar o placar ou fechar o jogo;
- adicione um passo antes de começar a partida para que o usuário digite o nome;
- armazene o número de *chutes* por partida;
- imprima o placar quando solicitado;
- imprima o placar em ordem crescente pelo número de *chutes*.
